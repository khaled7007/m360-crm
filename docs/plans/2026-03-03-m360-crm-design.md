# M360 — SME Lending CRM Design

## Overview

M360 is a full-lifecycle CRM for managing Shariah-compliant (Murabaha) SME lending in Saudi Arabia. Built for internal loan officers and operations staff (20-100 users), it covers lead acquisition through post-disbursement servicing including collections.

**Key characteristics:**
- Islamic finance (Murabaha) product focus
- Saudi regulatory context (SAMA)
- Arabic + English bilingual with RTL support
- Deep integration with Saudi identity and credit services

## Architecture

**Approach:** Modular monolith — single Go backend with domain-separated packages, Next.js frontend.

## Domain Model

### Organizations (SME Borrowers)

- **Organization:** CR number (Watheq-verified), trade name (AR/EN), industry, legal structure, founding date, address, Bayan credit data, financial summary
- **Contact:** Linked to organization — name, role (owner/CFO/authorized signatory/guarantor), national ID, Nafath verification status, SIMAH score, phone, email

### Lead & Pipeline

- **Lead:** Source (referral, marketing, walk-in, website), initial contact info, financing need estimate, assigned loan officer, qualification status
- **Opportunity:** Qualified lead with specific Murabaha request — requested amount, desired term, purpose, expected close date, pipeline stage

### Application

The central entity. Stages:

```
Draft → Submitted → Pre-Approved → Documents Collected → Credit Assessment → Committee Review → Approved/Rejected → Disbursed
```

Links to: organization, contacts (guarantors), product template, required documents checklist, credit assessment report, committee decision.

### Product Templates

Murabaha template: cost price, profit margin, payment frequency, maximum tenor, min/max amount, eligibility criteria, fee schedule, required document checklist.

### Facility (Post-Disbursement)

Active Murabaha contract: principal, profit amount, total payable, repayment schedule, outstanding balance, payment history, delinquency status, collections status.

### Activities & Communication

Activity log on every entity: calls, emails, meetings, notes, tasks, SMS sent. Document management: upload, version, categorize, link to application/entity.

## Key Workflows

### Lead-to-Disbursement Pipeline

1. **New Lead** — Loan officer qualifies the lead
2. **Opportunity Created** — Specific Murabaha financing need identified
3. **Pre-Approval** — Quick credit check (SIMAH/Bayan), Nafath verification, Watheq CR check
4. **Pre-Approved** — Data Entry collects full document set
5. **Documents Complete** — Credit Analyst performs full assessment, produces risk score and financial analysis report
6. **Compliance Cleared** — Compliance Officer reviews AML/KYC
7. **Committee Package** — Submitted for credit committee review
8. **Committee Decision** — Members vote (approve/reject/conditions)
9. **Approved** — Murabaha contract generated, presented to borrower
10. **Disbursed** — Facility created, repayment schedule generated

### Credit Committee Workflow

- Credit Analyst prepares committee package (financial analysis, credit report, recommendation)
- Package submitted for committee review
- Each committee member reviews and votes: Approve / Reject / Approve with Conditions
- Configurable quorum rules
- Decision recorded with individual votes and comments
- If conditions: tracked as action items that must be cleared before disbursement

### Collections Workflow

- System flags overdue payments based on repayment schedule
- Auto-assigns to Collections Officer
- Escalation ladder: reminder SMS → phone call → formal notice → restructuring proposal
- Collections Officer logs all actions as activities
- Manager dashboard shows Portfolio at Risk (PAR) metrics

## Roles & Permissions

7 roles with the following access matrix:

| Role | Lead | Opportunity | Application | Committee | Facility | Collections | Reports | Admin |
|------|------|------------|-------------|-----------|----------|-------------|---------|-------|
| Admin | Full | Full | Full | Full | Full | Full | Full | Full |
| Manager | View all | View all | View all | Vote + oversee | View all | Oversee | Full | Config |
| Loan Officer | CRUD own | CRUD own | Create, view own | View decisions | View own | - | Own portfolio | - |
| Credit Analyst | View | View | Assess + recommend | Prepare package | View | - | Risk reports | - |
| Compliance | - | View | Review compliance | View | View | - | Compliance reports | - |
| Collections | - | - | View | - | View | CRUD | Collections reports | - |
| Data Entry | Create | - | Upload docs, data entry | - | - | - | - | - |

"Own" means records assigned to that user. Managers see all records.

## Saudi Service Integrations

Each integration follows an adapter pattern: real implementation + mock fallback for development.

| Service | Purpose | When Called |
|---------|---------|------------|
| Nafath | Identity verification | Pre-approval — verify authorized signatories |
| Yaqeen | Identity data enrichment | Pre-approval — pull citizen/resident data |
| SIMAH | Consumer credit score | Pre-approval — check individual guarantors/owners |
| Bayan | Commercial credit report | Pre-approval — pull business credit history |
| Watheq | CR verification | Lead qualification — verify business registration is active |

## Analytics & Dashboards

### Manager Dashboard
Pipeline funnel, conversion rates, disbursement volume (monthly/quarterly), portfolio at risk (PAR30/60/90), officer performance scorecards, average processing time per stage.

### Loan Officer Dashboard
My pipeline (leads, opportunities, applications), tasks due today, pending document requests, upcoming repayment alerts.

### Collections Dashboard
Overdue accounts by aging bucket, collection rate, escalation status, upcoming actions.

### Executive Reports
Total portfolio size, new disbursements vs. target, delinquency trends, product performance, yield analysis.

## Notifications

- **In-app:** Real-time via WebSocket — new assignments, stage changes, committee decisions, overdue alerts
- **Email:** Application status changes, committee invitations, weekly summary reports
- **SMS:** Payment reminders to borrowers (7 days before, on due date, overdue)

## Required Documents (Standard Set)

- Commercial Registration (CR)
- Financial statements (2-3 years)
- Bank statements (6-12 months)
- National ID copies (owner + guarantors)
- Board resolution / authorization letter
- Company profile
- Tax registration certificate (Zakat)
